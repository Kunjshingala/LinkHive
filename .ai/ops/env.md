# Environment & Configuration — LinkHive

## Firebase Configuration

Firebase config is auto-generated by `flutterfire configure`. **Never edit manually.**

| File | Purpose |
|---|---|
| `lib/firebase_options.dart` | Auto-generated Firebase options (platform-specific) |
| `android/app/google-services.json` | Android Firebase config — gitignored, do not commit |
| `ios/Runner/GoogleService-Info.plist` | iOS Firebase config — gitignored, do not commit |
| `firebase.json` | Firebase CLI project config (safe to commit) |

---

## Env Variable Keys

This project does **not** currently use a `.env` file. Firebase configuration is handled through the SDK config files above.

If you need to add env vars in the future, use the `--dart-define` build flag pattern:

```bash
fvm flutter run --dart-define=MY_KEY=value
```

And access in code:
```dart
const myKey = String.fromEnvironment('MY_KEY');
```

**Never hardcode env values in source files.**

---

## Flutter Version (FVM)

Flutter version is pinned via FVM. Check `.fvmrc` for the active version. Always use FVM-prefixed commands:

```bash
fvm flutter --version   # check active version
fvm use <version>       # switch version
```

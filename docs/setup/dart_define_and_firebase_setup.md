## What is `--dart-define`?

`--dart-define` is a Flutter build flag that lets you pass **key-value pairs** into your Dart code at **compile time**. Think of it like environment variables, but they get baked into the app binary during the build.

### Single value

```bash
# Pass one value
fvm flutter run --dart-define=API_KEY=abc123
```

In Dart, you read it with:

```dart
const apiKey = String.fromEnvironment('API_KEY');
// apiKey == 'abc123'
```

### Multiple values from a file

When you have many values (like Firebase config), passing them one by one is impractical. That's where `--dart-define-from-file` comes in:

```bash
# Pass all values from a JSON file
fvm flutter run --dart-define-from-file=firebase_config.json
```

This reads every key-value pair from the JSON file and makes them all available via `String.fromEnvironment()` in Dart.

### Key Properties

| Property | Detail |
|----------|--------|
| **When are values injected?** | At compile time (not runtime) |
| **Can values change at runtime?** | No — they're `const` and baked into the binary |
| **Are values visible in the source code?** | No — they exist only in the JSON file and the compiled binary |
| **Can you use them in native code?** | Yes — they're available in Android's `BuildConfig` and iOS build settings |

## Why Not Use `google-services.json` / `GoogleService-Info.plist`?

The traditional Firebase setup uses:
- `android/app/google-services.json` (Android)
- `ios/Runner/GoogleService-Info.plist` (iOS)

These are generated by the Firebase Console and contain your API keys, app IDs, and project info.

### The problem

These files contain **secrets** (API keys) and are platform-specific. If you commit them to git:
- Anyone with repo access can see your Firebase API keys
- You need separate files for different environments (dev, staging, production)
- Two different files to manage for the same config

### The `--dart-define-from-file` advantage

| Aspect | Traditional (`google-services.json`) | Dart Define (`firebase_config.json`) |
|--------|--------------------------------------|--------------------------------------|
| Secret safety | File must be committed or manually shared | Gitignored — stays local |
| Platform files | 2 files (Android + iOS) | 1 JSON file for both |
| Environment switching | Swap files manually | Swap JSON file or use different file path |
| CI/CD | Need to decode/write files in pipeline | Pass values as environment variables |
| Native plugins needed | `com.google.gms.google-services` Gradle plugin | None — pure Dart |

## How It Works in LinkHive

Here's the complete flow from JSON file to Firebase initialization:

```
firebase_config.json                    (your local file, gitignored)
        |
        | --dart-define-from-file flag
        v
Dart compile-time constants             (baked into binary)
        |
        | String.fromEnvironment('FB_PROJECT_ID')
        v
lib/firebase_options.dart               (FirebaseConfig class reads the values)
        |
        | FirebaseConfig.currentPlatform
        v
lib/main.dart                           (Firebase.initializeApp)
        |
        v
Firebase SDK initialized and ready
```

### Step-by-step

1. You create `firebase_config.json` with your Firebase values
2. You run `fvm flutter run --dart-define-from-file=firebase_config.json`
3. Flutter compiler reads the JSON and injects each key-value pair
4. `lib/firebase_options.dart` defines `FirebaseConfig` class that reads these values using `String.fromEnvironment()`
5. `lib/main.dart` calls `Firebase.initializeApp(options: FirebaseConfig.currentPlatform)`
6. `FirebaseConfig.currentPlatform` returns the correct `FirebaseOptions` based on whether it's Android or iOS

## Firebase Project Setup (From Scratch)

If you're setting up LinkHive for the first time and don't have a Firebase project, follow these steps.

### Step 1: Create a Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Click **"Create a project"** (or "Add project")
3. Enter a project name (e.g., `LinkHive`)
4. Enable or disable Google Analytics (either is fine)
5. Click **Create project** and wait for it to initialize

### Step 2: Register an Android App

1. In the Firebase Console, click **"Add app"** > **Android**
2. Enter the package name: `com.link.hive`
3. Enter a nickname: `LinkHive Android`
4. **Skip** the SHA-1 for now (we'll add it for Google Sign-In later)
5. Click **Register app**
6. **Skip** downloading `google-services.json` — we don't use it
7. **Skip** the "Add Firebase SDK" steps — already in `pubspec.yaml`
8. Click **Continue to console**

### Step 3: Register an iOS App

1. Click **"Add app"** > **iOS**
2. Enter the bundle ID: `com.link.hive`
3. Enter a nickname: `LinkHive iOS`
4. Click **Register app**
5. **Skip** downloading `GoogleService-Info.plist` — we don't use it
6. **Skip** the remaining steps
7. Click **Continue to console**

### Step 4: Enable Authentication

1. In the left sidebar, click **Authentication**
2. Click **"Get started"**
3. Go to the **Sign-in method** tab
4. Enable **Email/Password**:
   - Click on it > Toggle **Enable** > Save
5. Enable **Google**:
   - Click on it > Toggle **Enable**
   - Select a support email
   - Click **Save**
   - **Important:** After enabling Google sign-in, note the **Web client ID** shown under "Web SDK configuration" — you'll need this as `FB_ANDROID_CLIENT_ID`

### Step 5: Set Up Firestore

1. In the left sidebar, click **Firestore Database**
2. Click **"Create database"**
3. Choose a location closest to your users
4. Start in **test mode** (allows all reads/writes for 30 days — fine for development)
5. Click **Create**

### Step 6: Get Your Config Values

Now collect all the values you need for `firebase_config.json`.

**Project Settings (shared values):**
1. Click the gear icon > **Project Settings** > **General** tab
2. Note down:
   - **Project ID** → `FB_PROJECT_ID`
   - **Project number** → `FB_MESSAGING_SENDER_ID` (yes, the project *number*)
   - **Storage bucket** → `FB_STORAGE_BUCKET` (looks like `your-project.firebasestorage.app`)

**Android app values:**
1. Still in Project Settings, scroll to **"Your apps"** section
2. Click on your Android app
3. Note down:
   - **API key** → `FB_API_KEY_ANDROID`
   - **App ID** → `FB_APP_ID_ANDROID` (looks like `1:123456:android:abcdef`)

**iOS app values:**
1. Click on your iOS app
2. Note down:
   - **API key** → `FB_API_KEY_IOS`
   - **App ID** → `FB_APP_ID_IOS` (looks like `1:123456:ios:abcdef`)
   - **Bundle ID** → `FB_IOS_BUNDLE_ID` (should be `com.link.hive`)

**Google Sign-In values:**
1. Go to **Authentication** > **Sign-in method** > **Google**
2. Expand **"Web SDK configuration"**
3. Note the **Web client ID** → `FB_ANDROID_CLIENT_ID`
   - This is used as the `serverClientId` for Google Sign-In on Android
4. For `FB_IOS_CLIENT_ID`:
   - Go to [Google Cloud Console](https://console.cloud.google.com/) > **APIs & Services** > **Credentials**
   - Find the **iOS client** OAuth 2.0 Client ID
   - Copy the **Client ID** → `FB_IOS_CLIENT_ID`

### Step 7: Add SHA-1 for Google Sign-In (Android)

Google Sign-In on Android requires your app's SHA-1 fingerprint.

**Debug SHA-1:**
```bash
# From the project root
cd android && ./gradlew signingReport
```

Look for the `SHA1` value under the `debug` variant.

**Add to Firebase:**
1. Go to Firebase Console > Project Settings > Your Apps > Android app
2. Click **"Add fingerprint"**
3. Paste the SHA-1 value
4. Click **Save**

## Creating Your Config File

1. Copy the example file:
   ```bash
   cp firebase_config.json.example firebase_config.json
   ```

2. Open `firebase_config.json` and fill in your values:

   ```json
   {
     "_comment_shared": "── Shared (Firebase Console > Project Settings > General) ──",
     "FB_PROJECT_ID": "your-project-id",
     "FB_MESSAGING_SENDER_ID": "123456789",
     "FB_STORAGE_BUCKET": "your-project.firebasestorage.app",

     "_comment_android": "── Android (Firebase Console > Project Settings > Your Apps > Android) ──",
     "FB_API_KEY_ANDROID": "AIzaSy...",
     "FB_APP_ID_ANDROID": "1:123456789:android:abcdef123456",

     "_comment_ios": "── iOS (Firebase Console > Project Settings > Your Apps > iOS) ──",
     "FB_API_KEY_IOS": "AIzaSy...",
     "FB_APP_ID_IOS": "1:123456789:ios:abcdef123456",
     "FB_IOS_BUNDLE_ID": "com.link.hive",

     "_comment_google_sign_in": "── Google Sign-In ──",
     "FB_ANDROID_CLIENT_ID": "123456789-xxxxx.apps.googleusercontent.com",
     "FB_IOS_CLIENT_ID": "123456789-yyyyy.apps.googleusercontent.com"
   }
   ```

3. **Verify** that `firebase_config.json` is in `.gitignore`:
   ```bash
   grep firebase_config.json .gitignore
   ```
   You should see it listed. **Never commit this file.**

## Running the App

### Terminal

```bash
# Debug mode
fvm flutter run --dart-define-from-file=firebase_config.json

# Release mode
fvm flutter run --release --dart-define-from-file=firebase_config.json

# Build APK
fvm flutter build apk --release --dart-define-from-file=firebase_config.json

# Build iOS
fvm flutter build ios --release --no-codesign --dart-define-from-file=firebase_config.json
```

The `--dart-define-from-file` flag must be passed **every time** you run or build. Without it, all Firebase config values will be empty strings and the app will crash on startup.

### VS Code

Pre-configured in `.vscode/launch.json` — just press **F5** or click **Run > Start Debugging**. The `toolArgs` already include the `--dart-define-from-file` flag:

```json
{
    "name": "linkhive",
    "request": "launch",
    "type": "dart",
    "toolArgs": ["--dart-define-from-file=firebase_config.json"]
}
```

Three configurations are available:
- **linkhive** — debug mode
- **linkhive (profile mode)** — performance profiling
- **linkhive (release mode)** — production build

### Android Studio

Pre-configured in `.run/linkhive.run.xml` — select **linkhive** from the run configuration dropdown and click **Run** (Shift+F10).

The config includes:
```xml
<option name="additionalArgs" value="--dart-define-from-file=firebase_config.json" />
```

## How the Code Reads These Values

The bridge between the JSON file and Firebase SDK is `lib/firebase_options.dart`:

```dart
abstract class FirebaseConfig {
  // String.fromEnvironment reads compile-time --dart-define values
  static const projectId = String.fromEnvironment('FB_PROJECT_ID');
  static const messagingSenderId = String.fromEnvironment('FB_MESSAGING_SENDER_ID');
  static const storageBucket = String.fromEnvironment('FB_STORAGE_BUCKET');

  // Platform-specific values
  static const androidApiKey = String.fromEnvironment('FB_API_KEY_ANDROID');
  static const androidAppId = String.fromEnvironment('FB_APP_ID_ANDROID');
  static const androidClientId = String.fromEnvironment('FB_ANDROID_CLIENT_ID');

  static const iosApiKey = String.fromEnvironment('FB_API_KEY_IOS');
  static const iosAppId = String.fromEnvironment('FB_APP_ID_IOS');
  static const iosBundleId = String.fromEnvironment('FB_IOS_BUNDLE_ID');
  static const iosClientId = String.fromEnvironment('FB_IOS_CLIENT_ID');

  // Returns the right FirebaseOptions based on the current platform
  static FirebaseOptions get currentPlatform {
    switch (defaultTargetPlatform) {
      case TargetPlatform.android:
        return android;
      case TargetPlatform.iOS:
        return ios;
      default:
        throw UnsupportedError('Unsupported platform');
    }
  }

  static const android = FirebaseOptions(
    apiKey: androidApiKey,
    appId: androidAppId,
    messagingSenderId: messagingSenderId,
    projectId: projectId,
    storageBucket: storageBucket,
  );

  static const ios = FirebaseOptions(
    apiKey: iosApiKey,
    appId: iosAppId,
    messagingSenderId: messagingSenderId,
    projectId: projectId,
    storageBucket: storageBucket,
    iosBundleId: iosBundleId,
  );
}
```

### How `String.fromEnvironment` works

```dart
static const projectId = String.fromEnvironment('FB_PROJECT_ID');
```

- This is a **compile-time** constant, not a runtime call
- The Dart compiler replaces it with the actual value from `--dart-define`
- If the key isn't provided, it returns an empty string `''` (or a custom default)
- Because it's `const`, the value is embedded in the compiled binary — it cannot be changed after build

### Where each value is used

| Config Key | Used In | Purpose |
|------------|---------|---------|
| `FB_PROJECT_ID` | `FirebaseConfig` | Identifies your Firebase project |
| `FB_MESSAGING_SENDER_ID` | `FirebaseConfig` | Required by Firebase SDK initialization |
| `FB_STORAGE_BUCKET` | `FirebaseConfig` | Firebase Storage URL |
| `FB_API_KEY_ANDROID` | `FirebaseConfig.android` | Android API key for Firebase services |
| `FB_APP_ID_ANDROID` | `FirebaseConfig.android` | Android app registration ID |
| `FB_API_KEY_IOS` | `FirebaseConfig.ios` | iOS API key for Firebase services |
| `FB_APP_ID_IOS` | `FirebaseConfig.ios` | iOS app registration ID |
| `FB_IOS_BUNDLE_ID` | `FirebaseConfig.ios` | iOS bundle identifier |
| `FB_ANDROID_CLIENT_ID` | `AuthService` | Google Sign-In `serverClientId` (Android) |
| `FB_IOS_CLIENT_ID` | `AuthService` | Google Sign-In `clientId` (iOS) |

## Troubleshooting

| Issue | Cause | Solution |
|-------|-------|----------|
| `No Firebase App '[DEFAULT]' has been created` | Missing `--dart-define-from-file` flag | Always pass `--dart-define-from-file=firebase_config.json` when running |
| App crashes on startup with empty Firebase config | `firebase_config.json` has empty values | Fill in all values from your Firebase Console |
| `firebase_config.json` not found | Wrong working directory | Run commands from the project root (`LinkHive/Code/LinkHive/`) |
| Google Sign-In fails on Android | Missing SHA-1 fingerprint | Run `./gradlew signingReport` and add SHA-1 to Firebase Console |
| Google Sign-In fails on iOS | Wrong `FB_IOS_CLIENT_ID` or missing URL scheme | Check the reversed client ID URL scheme in `ios/Runner/Info.plist` |
| `_comment_*` keys in JSON | Ignored by Dart — they're just for readability | No action needed — these are documentation comments |
| Values work in debug but not release | Different signing key = different SHA-1 | Add your release SHA-1 to Firebase Console too |
| `PlatformException` on `Firebase.initializeApp` | Wrong API key or app ID | Double-check values match exactly what Firebase Console shows |

### Quick Verification

To verify your config is loaded correctly, add a temporary debug print:

```dart
// In main.dart, before Firebase.initializeApp:
debugPrint('Project ID: ${FirebaseConfig.projectId}');
debugPrint('Android API Key: ${FirebaseConfig.androidApiKey}');
```

If these print empty strings, the `--dart-define-from-file` flag isn't being passed correctly.
